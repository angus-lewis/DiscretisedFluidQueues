Models and Discretisations

Model-related Types

DiscretisedFluidQueues.Phase — Type
Phase

Represents a phase of the fluid queue.

Arguments:

  • c::Float64: the rate of change of the fluid X(t) associated with the phase
  • m::Int64: either -1 or 1, membership of the phase with either the set of positive or negative phases. Default is positive, i.e. 1. Only relevant for FRAPApproximation.
  • lpm::Bool: whether the phase has a point mass at the left/lower boundary
  • rpm::Bool: whether the phase has a point mass at the right/upper boundary

DiscretisedFluidQueues.DiscretisedFluidQueue — Type
DiscretisedFluidQueue{T <: Mesh}

A container to hold a FluidQueue and a discretisation scheme (a mesh). Parametric type depending on the type of discretisation scheme (<:Mesh)

Arguments:

  • model::Model: a FluidQueue
  • mesh::T: either a DGMesh, FVMesh or FRAPMesh specifying the discretisation scheme for the continuous level of the fluid queue, X(t).

DiscretisedFluidQueues.DGMesh — Type
DGMesh <: Mesh

A structure representing a discretisation scheme to be used for a DiscretisedFluidQueue.

Arguments:

  • nodes::Array{<:Real, 1}: The edges of the cells.
  • n_bases::Int: The number of basis functions used to represent the solution on each cell
source
DiscretisedFluidQueues.FRAPMesh — Type
FRAPMesh <: Mesh

A QBD-RAP (FRAM) discretisation scheme for a DiscretisedFluidQueue.

Arguments

  • nodes: the cell edges
  • me: the MatrixExponential used to approximate model the fluid queue on each cell.
source
DiscretisedFluidQueues.FVMesh — Type
FVMesh <: Mesh

A finite volume discretisation scheme for a DiscretisedFluidQueue.

Arguments

  • nodes: the cell edges (the nodes are at the center of the cell)
  • order: the order of the polynomial interpolation used to approximate the flux at the

cell edges.
